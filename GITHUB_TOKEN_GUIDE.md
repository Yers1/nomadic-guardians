# Как создать GitHub Personal Access Token

## Зачем нужен токен?

GitHub больше не принимает обычные пароли при работе с git через HTTPS. Вместо этого нужно использовать **Personal Access Token (PAT)**.

---

## Пошаговая инструкция:

### 1. Войдите в GitHub
- Перейдите на https://github.com
- Войдите в свой аккаунт

### 2. Откройте настройки
- Нажмите на ваш **аватар** (правый верхний угол)
- Выберите **Settings**

### 3. Перейдите в Developer settings
- Прокрутите вниз в левом меню
- Нажмите **Developer settings** (в самом низу)

### 4. Создайте токен
- В левом меню выберите **Personal access tokens**
- Выберите **Tokens (classic)**
- Нажмите **Generate new token**
- Выберите **Generate new token (classic)**

### 5. Настройте токен
Заполните форму:

- **Note** (название): 
  - Например: "Vercel Deploy" или "Local Development"
  - Это просто название для вашего удобства

- **Expiration** (срок действия):
  - Выберите срок (например, 90 days, 1 year, или No expiration)
  - Рекомендую: 90 days для безопасности

- **Scopes** (права доступа):
  - Отметьте галочку **repo** (полный доступ к репозиториям)
  - Это даст права на чтение и запись в репозитории

### 6. Создайте токен
- Нажмите **Generate token** (внизу страницы)
- ⚠️ **ВАЖНО**: GitHub покажет токен только ОДИН РАЗ!
- Токен выглядит так: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
- **Скопируйте токен СРАЗУ** и сохраните в безопасном месте

### 7. Используйте токен
При выполнении `git push`:
- **Username**: ваш GitHub username
- **Password**: вставьте токен (НЕ обычный пароль!)

---

## Пример использования:

```bash
git push -u origin main

# Когда попросит:
Username for 'https://github.com': ваш-username
Password for 'https://ваш-username@github.com': ghp_ваш-токен-здесь
```

---

## Безопасность:

✅ **Хорошо:**
- Храните токен в безопасном месте
- Используйте разные токены для разных целей
- Устанавливайте срок действия токена

❌ **Плохо:**
- Не делитесь токеном публично
- Не коммитьте токен в git
- Не используйте один токен везде

---

## Альтернатива: SSH ключи

Если не хотите использовать токены, можно настроить SSH:

1. Создайте SSH ключ: `ssh-keygen -t ed25519 -C "your_email@example.com"`
2. Добавьте ключ в GitHub: Settings → SSH and GPG keys
3. Используйте SSH URL: `git@github.com:USERNAME/repo.git`

Но для быстрого деплоя токен проще!

---

## Если забыли токен:

1. Перейдите в Settings → Developer settings → Personal access tokens
2. Найдите ваш токен в списке
3. Вы можете его удалить или создать новый
4. Но увидеть старый токен снова нельзя - нужно создать новый
